We are building the prod grade enterpise devops project, which included all the tools like github, terraform, docker, k8s, istio, argocd etc main things and if any start up wanted to deploy their applciation in the production they jus clone this project and add their required configurations thats it , new env ready 

this projecy starts with postgress database, account service, transaction service, notification service, api gateway and frontend 6 microservices we are dockerising and deploying 

then we will go for the infra provisioning to deploy our app in this production

the postgress is a database where we will update all the data created by applciation like 
1. creating the users
2. creating the tables
3. creating the accounts with adding the money inside their accounts
4. creating the transactions between accounts like debiting, crediting etc
5. we store all the data generated by application

2nd microservice is account service it act like a accountant who update all the resords in the balancesheet same like this microservice is 
1. checkign the accounts
2. checking the balance of the user accounts inside the tables
3. it will list out all the data inside the database 

3rd Microservice is Account Service - which is act like a agent, servent which helps us to do the transactions between users

4th notification service: it is like a communicator which will alert us for each transaction if it is succes, fail, not found , insufficiant etc

5th api gateway: this is single most imp microservice
